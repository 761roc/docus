# Vue 注意事项与常见坑

## 1. 解构导致响应式丢失

不要直接解构 `reactive` 对象字段；需要时使用 `toRefs`。

## 2. `watch` 监听范围过大

深层监听会带来性能开销，优先监听关键字段。

## 3. 列表 `key` 不稳定

避免使用数组索引作为长期稳定列表的 `key`。

## 4. 副作用未清理

在 `onBeforeUnmount` 中清理 `setInterval`、事件监听和第三方实例。

## 5. 全局状态滥用

- 页面私有状态放在组件内。
- 只有跨页面、跨模块共享状态再放 Pinia。

## 6. 组合式逻辑缺乏边界

`composables` 应按单一职责拆分，避免一个函数承担所有业务。
